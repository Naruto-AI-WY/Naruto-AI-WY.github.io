Mac下使用Hugo+Github托管搭建个人博客

2020-03-10
pages
约 2205 字 预计阅读 5 分钟

文章目录

Hugo养成记

疫情期间，时间比较多，一直想着写点自己的点滴，也没什么特别的主题，当做是日常记录吧，具体后面写点啥，后面慢慢开脑洞吧。第一篇就留给博客网站搭建纪实了。

安装Hugo：

Hugo号称世界上最快的建站框架，由go语言开发。 Mac下直接使用 Homebrew 安装：

1
2
3
4
5

brew install hugo
# 检查安装成功  
hugo version 

Hugo Static Site Generator v0.66.0/extended darwin/amd64 BuildDate: unknown

注意：如果发现使用brew命令，却总是提示：-bash: brew: command not found，则是你的mac上homebrew没有安装

1

/usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

建立个人博客目录：

1
2
3
4
5
6

hugo new site your_blog_name  # 此处your_blog_name会生成一个同名的文件夹，里面是此次的project site文件
cd your_blog_name  
git init  #初始化本地数字仓库

# 目录结构，tree命令 
tree blog
  • config.toml 是配置文件，在里面可以定义博客地址、构建配置、标题、导航栏等等。
  • themes 为放置主题的文件夹，可以在themes.gohugo.io 查看喜欢的主题。另外如果更改了已应用的主题的src文件夹下的内容，需要重新build主题
  • content 是存放博客内容的文件，支持任何级别的文件夹和文件的嵌套，最后的文件系统里 markdown 文件路径和站点 url 的对应关系一般如下：
.
└── content
    └── about
    |   └── _index.md  // <- https://example.com/about/
    ├── post
    |   ├── firstpost.md   // <- https://example.com/post/firstpost/
    |   ├── happy
    |   |   └── ness.md  // <- https://example.com/post/happy/ness/
    |   └── secondpost.md  // <- https://example.com/post/secondpost/
    └── quote
        ├── first.md       // <- https://example.com/quote/first/
        └── second.md      // <- https://example.com/quote/second/

有两个文件名名字是有特殊含义的： _index.md 和 index.md 可以用于组织页面，更多请参考 Hugo 官方文档。

  • static包括了 css, js, fonts, media 等，决定网站的外观
  • archetypes包含了default.md文件，是新建博客文件时使用的模板，可以custom模板。

下载主题

这里我选择了even主题，可以根据喜好选择。even主题
把github上的主题直接clone到你的博客文件夹的themes目录下，这样对主题的改动也可以同时提交到git控制中。

git clone https://github.com/olOwOlo/hugo-theme-even themes/even

也可以添加到 git 的 submodule 中，优点是后面讲到用 travis 自动部署时比较方便。如果需要对主题做更改，最好fork主题再做改动。

git submodule add https://github.com/olOwOlo/hugo-theme-even.git themes/even

注：如果git clone不成功或者速度过慢的话，可以考虑搜索一下更改hosts文件的方法或者使用代理服务器，可自行搜索

重新build主题

更改了主题的话，需要切换到 themes/even 目录下重新 build。

cd themes/even && npm i && npm start

开始写文章

hugo new post/my-first-post.md

文章顶部是一些 meta 信息，可以更改。也可以使用一些固定模板自动生成 meta 信息，可以使用编辑器更改你的博客文件目录下的archetypes文件夹里面的default.md的文件

配置项目

拷贝themes/even/exampleSite下的config.toml覆盖根目录下的config.toml，设置config.toml中的一些值。

logoTitle = "你的首页标题名"  

# 归档、标签、分类每页显示的文章数目，建议修改为一个较大的值  
archivePaginate = 5  

# 是否在归档页显示文章的总数  
showArchiveCount = true  

# 是否使用mathjax（数学公式）  
mathjax = true      # see https://www.mathjax.org/   
mathjaxEnableSingleDollar = true   # 是否使用 $...$ 即可進行inline latex渲染  
mathjaxEnableAutoNumber = false   # 是否使用公式自动编号  

# 链接到markdown原始文件（仅当允许hugo生成markdown文件时有效）  
linkToMarkDown = false    # Only effective when hugo will output .md files.  

# 启用公共CDN，需自行定义  
[params.publicCDN]        # load these files from public cdn          

# 社交链接                
[params.social]

启动博客的本地预览

在站点的根目录下使用命令进行本地启动，本地启动的命令如下：

hugo server -D

执行成功的话，就可以在浏览器中输入http://localhost:1313/ 在本地预览博客。

GitHub 配置

在github上新建一个项目，注意项目的名称必须是（你的用户名.github.io)

设置Git的user name和email

git config --global user.name "Your username"  
git config --global user.email "your email@xx.com"

生成密钥

ssh-keygen -t rsa -f ~/.ssh/id_rsa_blog  -C "your email@xx.com"

将ssh公钥内容拷贝到Github

cat ~/.ssh/id_rsa_blog.pub  

Your identification has been saved in id_rsa_blog.  
Your public key has been saved in id_rsa_blog.pub.

或者编辑器打开~/.ssh/id_rsa_blog.pub文件，copy其中的内容（RSA公钥），然后网页打开你的github repo，然后进入Setting->Deploy keys，将刚才copy的内容添加进去。添加成功后，如下图所示

部署到Github个人页面

在Github创建一个仓库，例如名字叫blog，可以是私有的，这个仓库用来存放网站内容和源文件. 再创建一个名称为username.github.io的仓库，username为GitHub用户名，这个仓库用于存放最终发布的网站内容，进入本地网站目录

cd <YOUR lOCAL BLOG> 
git remote add origin git@github.com:username/username.github.io.git  
#将本地Local Blog仓库关联到github远程仓库

然后删除本地网站目录下的public文件夹

rm -rf ./public

创建public子模块

git submodule add -b master git@github.com:<USERNAME>/<USERNAME>.github.io.git public

最后执行hugo命令，生成网站静态内容到public文件夹，然后push到远程github repo仓库。

hugo -t even  

cd public  
git status  
git add .  #添加文件到缓冲
git commit -m "first commit"  
git push -u origin master

自动部署脚本

将以下命令添加到自动执行脚本中deploy.sh，脚本commit提交信息会使用执行时的时间，将脚本放到网站项目路径下，写完博客后，双击运行即可自动部署发布

#!/bin/bash  

echo -e "\033[0;32mDeploying updates to GitHub...\033[0m"  

# Build the project.  
hugo -t even # if using a theme, replace with `hugo -t <YOURTHEME>`  

# Go To Public folder  
cd public  
# Add changes to git.  
git add .  

# Commit changes.  
msg="rebuilding site `date`"  
if [ $# -eq 1 ]  
  then msg="$1"  
fi  
git commit -m "$msg"  

# Push source and build repos.  
git push origin master  

# Come Back up to the blog local Project Root  
cd ..
