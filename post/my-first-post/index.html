GitHub 配置

在github上新建一个项目，注意项目的名称必须是（你的用户名.github.io)

Git的配置说明

Git 自带一个 git config 的工具来帮助设置控制 Git 外观和行为的配置变量。 这些变量存储在三个不同的位置：

  1. /etc/gitconfig 文件: 包含系统上每一个用户及他们仓库的通用配置。 如果在执行 git config 时带上 –system 选项，那么它就会读写该文件中的配置变量。 （由于它是系统配置文件，因此你需要管理员或超级用户权限来修改它。）

  2. ~/.gitconfig 或 ~/.config/git/config 文件：只针对当前用户。 你可以传递 –global 选项让 Git 读写此文件，这会对你系统上 所有 的仓库生效。

  3. 当前使用仓库的 Git 目录中的 config 文件（即 .git/config）：针对该仓库。 你可以传递 –local 选项让 Git 强制读写此文件，虽然默认情况下用的就是它。。 （当然，你需要进入某个 Git 仓库中才能让该选项生效。）

每一个级别会覆盖上一级别的配置，所以 .git/config 的配置变量会覆盖 /etc/gitconfig 中的配置变量。

你可以通过以下命令查看所有的配置以及它们所在的文件：

1

git config --list --show-origin

设置Git的user name和email

在本机git全局变量上设置账号和email。

1
2

git config --global user.name "Your username"  
git config --global user.email "your email@xx.com"

如果使用 https 连接, 通常要求你输入 username & password. 如果想每次git指令登录不需要 username & password，可以采用采用如下

  • 采用 SSH 连接方式 前提当然你得有权限将自己主机 id_rsa.pub 加入到 git 账户的 SSH Keys 中
  • 采用 HTTPS 连接方式 只要你有仓库访问权限即可, 使用 credential 来进行配置

Git credential凭证存储

Git 拥有一个凭证系统credential.helper来处理这个账号和密码的事宜
Git 的选项如下：
默认所有都不缓存。 每一次连接都会询问你的用户名和密码。

  • “cache” 模式会将凭证存放在内存中一段时间。 密码永远不会被存储在磁盘中，并且在15分钟后从内存中清除。

  • “store” 模式会将凭证用明文的形式存放在磁盘中，并且永不过期。 这意味着除非你修改了你在 Git 服务器上的密码，否则你永远不需要再次输入你的凭证信息。 这种方式的缺点是你的密码是用明文的方式存放在你的 home 目录下。

  • 我使用的是Mac，Git 还有一种 “osxkeychain” 模式，它会将凭证缓存到你系统用户的钥匙串中。 这种方式将凭证存放在磁盘中，并且永不过期，但是是被加密的，这种加密方式与存放 HTTPS 凭证以及 Safari 的自动填写是相同的。

你可以设置 Git 的配置来选择上述的一种方式

例如全局使用cache存储方式。

1

$ git config --global credential.helper cache

钥匙链存储方式：

1

git  config --global  credential.helper  osxkeychain #钥匙链存储

查看当前的git配置

如果使用git命令时仍然有报错，就需要检查 当前的所有git config的配置信息。
首先运行下面的命令

1

git config --list

credential.helper=osxkeychain
user.name=xxxxx
user.email=xxxxx@xxxx.xx
core.repositoryformatversion=0
core.filemode=true
core.bare=false
core.logallrefupdates=true

检测用户名和邮箱和 push 时是不是一样，不一样的话，git push等命令肯定出错。至于为什么不使用配置的用户名呢，在于第一行的 credential.helper的取值，我们需要检测所有git config文件，查看其中定义的credential.helper的值。

只查询credential.helper的值的命令，可以用来检查当前的credential.helper的值的情况。

1

git config credential.helper #可用于检测当前的credential.helper的值

如果检查到了credential.helper已经被设置了值，则需要找到所以可以设置credential.helper的gitconfig文件，进行删除值。

定位gitconfig的credential.helper

一般 credential.helper 会配置在三个地方 local, global, system，分别用以下命令查看

1
2
3

git config --local credential.helper
git config --global credential.helper
git config --system credential.helper

如果找到了credential.helper=xxxx，删除其值。如果都没有找到credential.helper=xxxx，用如下的命令找出哪一个配置文件最后设置 credential.helper 的值：

1

git config --show-origin --get credential.helper

file:/Applications/Xcode.app/Contents/Developer/usr/share/git-core/gitconfig osxkeychain
原来 xcode 下有配置了 credential.helper，我们查看一下这个配置文件

1

cat /Applications/Xcode.app/Contents/Developer/usr/share/git-core/gitconfig

1
2

[credential]  
    helper = osxkeychain

Xcode也有gitconfig的配置文件,如果需要重新配置credential.helper，则需要更改已有的值。

  • 删除所有gitconfig的credential.helper的值
  • 根据自己的使用需求，使用cache/store/osxkeychain，具体的区别和定义请见参考资料。命令如下：
1

  git config --global credential.helper cache/store/osxkeychain

再次执行git push命令时，要求重新输入账号和密码，此时会根据前面配置的credential.helper的值进行存储。 设置成store（明码）和osxkeychain（钥匙链，加密）都会长期存储在硬盘文件上。

git-credential-store的搜索顺序

~/.git-credentials 用户特定的凭据文件。

$XDG_CONFIG_HOME/git/credentials 第二个用户特定的凭证文件。如果 $XDG_CONFIG_HOME 没有设置或为空， $HOME/.config/git/credentials 将被使用。如果 ~/.git-credentials 具有匹配的凭证，则不会使用存储在此文件中的任何凭证。如果你有时使用不支持它的老版本 Git，不建立这个文件是个好的选择。

对于凭证查找，将按照上面给出的顺序读取文件，发现第一个匹配的凭证优先于在列表下方的文件中找到的凭证。

凭证存储将默认写入列表中的第一个现有文件。如果这些文件都不存在，~/.git-credentials 将被创建并写入。

擦除证书时，匹配的证书将从所有文件中删除。

# Build the project.  
hugo -t even # if using a theme, replace with `hugo -t <YOURTHEME>`

# Come Back up to the blog local Project Root

  • git指令报错

这是我在使用过程中遇到的问题，安装完后一直没有关闭bash，一直可以正常的使用deploy程序push更改，但是重启了一次电脑后，再次打开bash，再次运行deploy自动上传更改时就报出了这个错误。搜索了相关的文章，了解到ssh-add 这个命令默认是把你指定的私钥进行高速缓存所以说当你重启之后，ssh-agent 服务也就重置了，再次push代码都必须重新ssh-add添加上私钥。

解决方案：

方案1 找到~/.ssh/config文件，如果没有此文件，可以自己创建。文件格式如下：

1
2
3
4

Host github.com
  Hostname github.com
  AddKeysToAgent yes
  IdentityFile ~/.ssh/id_rsa_blog

~/.ssh/config 就是一个配置文件，描述不同的秘钥对应的设置 —— 包括主机名、用户名、访问策略等等。此配置不局限于 Git，所有底层使用 SSH 的应用和命令都会遵照配置文件config的指示来找到对应的私钥，可以用来管理多组密钥对。 ~/.ssh/config 文件自动将密钥加载到 ssh-agent 中，并根据AddKeysToAgent和UseKeychain的值，通过ssh-agent或者Keychain存储密码。 对这个问题感兴趣可以参考文章：SSH Config 那些你所知道和不知道的事 和 是否必须每次添加ssh-add，以及将 SSH 密钥添加到 ssh-agent(github官方文档)
SSH 的配置文件（config file）有两个：

1
2

$ ~/.ssh/config            # 用户配置文件，如果你对文件夹里没有config文件，你可以自己建一个。
$ /etc/ssh/ssh_config      # 系统配置文件

~/.ssh/config是针对当前用户的配置，仅对一个用户有效。/etc/ssh/ssh_config是对本机所有用户。根据自己情况选择对应的配置文件修改。

方案2

Use keychain使用钥匙链 This was not the intended default behavior, so this has been changed in macOS 10.12.2. To store passphrases in the keychain, set this option in your ssh configuration file:

1

UseKeychain yes

This option can be set for all hosts, or for a subset of hosts as desired. For instance, you can configure a development machine as such: ~/.ssh/config

This will only store the passphrase in the keychain for that specific key
(如果你试图为GitHub执行这里操作)

1
2
3

Host github.com
    IdentityFile ~/.ssh/id_rsa #填写你的密码文件名
    UseKeychain yes

如果你想为每个主机启用它，只需添加：

1
2

Host *  
    UseKeychain yes
